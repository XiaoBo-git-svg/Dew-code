# Prompt 库使用说明

> 本文件是 Prompt 模板库的**格式规范**。当你需要将一份新的 Prompt 文档整理入库时，请严格按照本说明执行。

---

## 一、文件夹结构

```
Prompt/
├── README.md                          ← 你正在看的这份说明
├── Prompt-MOC.md                      ← 总索引（Map of Content）
├── 01-写作与内容创作/
│   ├── 01-文章大纲生成.md
│   ├── 02-文章润色.md
│   └── ...
├── 02-代码辅助开发/
├── 03-数据分析与总结/
├── 04-学习与教育/
├── 05-创意与设计/
├── 06-客服与问答/
└── 07-提示词技巧与方法/
```

### 现有分类（7 个）

| 编号 | 文件夹名 | 包含内容 |
|------|----------|----------|
| 01 | 写作与内容创作 | 文章、文案、邮件、简历等文字工作 |
| 02 | 代码辅助开发 | 代码生成、调试、重构、排查 |
| 03 | 数据分析与总结 | 文档摘要、数据解读、会议纪要 |
| 04 | 学习与教育 | 学习助手、概念解释、语言学习、翻译 |
| 05 | 创意与设计 | 头脑风暴、设计描述、故事创作 |
| 06 | 客服与问答 | 客服机器人等问答场景 |
| 07 | 提示词技巧与方法 | Prompt 编写方法论与技巧 |

### 分类规则

1. **优先归入现有分类** —— 大多数 Prompt 都能归入上述 7 个类别
2. **确实无法归类时，新建分类** —— 编号从 `08-` 开始递增，文件夹名格式为 `XX-分类名称`（中文）
3. **分类粒度适中** —— 不要为单个 Prompt 新建分类；一个分类下至少应有 2-3 个相关 Prompt

---

## 二、文件命名规范

- **格式**：`XX-中文名称.md`，`XX` 为两位数字编号
- **编号**：在所属文件夹内递增，从 `01-` 开始
- **名称**：简洁的中文名，3-6 个字为宜
- **示例**：`01-文章大纲生成.md`、`02-Bug调试.md`

---

## 三、Frontmatter 规范

每个 Prompt 文件**必须**包含以下 YAML frontmatter：

```yaml
---
title: "Prompt 标题"
category: "所属分类名（不带编号）"
difficulty: "入门"          # 或 "进阶"
tags:
  - prompt                  # 固定第一个 tag
  - 标签1                   # 2-4 个中文主题标签
  - 标签2
created: 2026-06-24         # 创建日期
source: "[[来源文件路径]]"    # 来源 wikilink
---
```

### 字段说明

| 字段 | 必填 | 类型 | 说明 |
|------|------|------|------|
| `title` | ✅ | string | 文件标题，与 H1 标题一致 |
| `category` | ✅ | string | 所属分类，如 `"写作与内容创作"`（不带 `01-` 前缀） |
| `difficulty` | ✅ | string | 只能是 `"入门"` 或 `"进阶"` |
| `tags` | ✅ | list | 第一个必须是 `prompt`，后面加 2-4 个主题标签 |
| `created` | ✅ | date | 创建日期，格式 `YYYY-MM-DD` |
| `source` | ✅ | string | 来源 wikilink，格式 `"[[Clippings/文件名]]"` |

## 四、正文结构模板

每个 Prompt 文件的正文**必须**按以下顺序组织：

```markdown
# 标题

> 一句话简介，说明这个 Prompt 的用途或核心理念。

---

## Prompt 模板

```
[这里是可复制的 Prompt 模板]
[用 [方括号] 标注需要用户填写的变量]
```

---

## 使用示例

```
[一个已经填写好变量的真实示例]
[展示 Prompt 实际使用时的完整样子]
```

---

## 技巧

- **技巧 1**：具体建议
- **技巧 2**：具体建议
- **技巧 3**：具体建议

---

## 相关模板

- [[Prompt/XX-分类/YY-名称|显示名称]]
- [[Prompt/XX-分类/YY-名称|显示名称]]
```

### 各部分要求

| 部分 | 必填 | 说明 |
|------|------|------|
| H1 标题 | ✅ | 与 frontmatter `title` 一致 |
| 引用简介 | ✅ | `>` 引用块，一句话概括用途 |
| Prompt 模板 | ✅ | 用代码块包裹，`[方括号]` 标注变量 |
| 使用示例 | ✅ | 至少一个填写完整的真实示例 |
| 技巧/原则 | ✅ | 3-5 条实用建议，加粗关键词 |
| 相关模板 | ✅ | 2-3 个 wikilink 指向关联文件 |
| 补充内容 | ❌ | 可选：速查表、对比表、流程图、检查清单等 |

## 五、Prompt 模板编写规范

### 变量标注

- 使用 `[方括号]` 标注用户需要替换的内容
- 变量名要直观易懂，用中文
- 示例：`[主题]`、`[目标读者]`、`[你的代码]`、`[期望输出格式]`

### 模板质量要求

1. **可直接复制使用** —— 用户复制模板后，只需替换变量即可发送
2. **指令明确** —— 用编号列表、分点说明，避免模糊表述
3. **给出约束** —— 指定输出格式、长度、风格等，减少 AI 自由发挥
4. **一次解决一个问题** —— 不要把多个不相关的任务塞进一个 Prompt

## 六、入库操作流程

当你拿到一份新的 Prompt 文档时，按以下步骤操作：

### Step 1：判断分类

```
新 Prompt 的核心用途是什么？
  → 匹配现有 7 个分类 → 进入该文件夹
  → 无法匹配 → 新建 XX-分类名 文件夹（编号递增）
```

### Step 2：确定编号

```
查看目标文件夹内已有文件
  → 新文件编号 = 最大编号 + 1
  → 示例：已有 01、02、03 → 新文件为 04
```

### Step 3：创建文件

按本说明的「正文结构模板」创建 `.md` 文件

### Step 4：更新 MOC

在 `Prompt-MOC.md` 中添加条目：

```markdown
### [[Prompt/XX-分类/新文件名|分类名]]
分类简介。

- [[Prompt/XX-分类/新文件名|Prompt 名称]]
```

- 如果分类已存在，在该分类的列表末尾追加一行
- 如果是新分类，在「分类索引」区域追加新的 `###` 段落

### Step 5：建立关联

- 在新文件的「相关模板」部分添加 2-3 个关联链接
- 检查关联文件是否也需要反向添加链接（可选）

## 八、常见问题

**Q：一个 Prompt 涉及多个分类怎么办？**
A：按**主要用途**归类。在「相关模板」中链接到其他分类的相关文件即可。

**Q：Prompt 来自网络，没有 Clippings 来源怎么办？**
A：`source` 字段填实际来源，如网页标题的 wikilink，或写 `"来自网络"`。尽量保留出处。

**Q：Prompt 太长，需要多个模板怎么办？**
A：在同一个文件中用 `### 子标题` 区分多个模板变体，每个都用代码块包裹。参考 `02-代码辅助开发/01-代码生成与解释.md`。

**Q：difficulty 选"入门"还是"进阶"？**
A：**入门** —— 任何人复制粘贴就能用；**进阶** —— 需要一定领域知识或需要多次调试优化。
