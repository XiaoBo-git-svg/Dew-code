# RAG 知识库问答

> 让 AI 基于你的私有知识库回答问题——先检索相关文档，再生成准确回答，解决 LLM 知识截止和幻觉问题。

---

## RAG 工作原理

```
用户提问 → 问题向量化 → 向量数据库检索 → 获取相关文档片段
                                                    ↓
                                    构建 Prompt（上下文 + 问题）→ LLM 生成回答
```

> RAG = Retrieval-Augmented Generation（检索增强生成）。没有 RAG，LLM 只知道训练数据里的内容；有了 RAG，LLM 就能用上你私有的、最新的数据。

---

## Prompt 模板

### 基础 RAG 问答模板

```
你是一个专业的 [领域] 助手。请基于以下上下文回答用户的问题。

规则：
1. 只基于提供的上下文回答，不要编造信息
2. 如果上下文中没有相关信息，请明确说"抱歉，我没有找到相关信息"
3. 回答要简洁准确，使用中文
4. 在回答中标注信息来源

上下文：
[检索到的文档片段]

用户问题：[用户的问题]

回答：
```

### 带引用来源的 RAG 模板

```
你是一个专业的 [领域] 知识库助手。请根据以下参考资料回答问题。

要求：
1. 严格基于参考资料回答，不要添加参考资料中没有的信息
2. 回答时必须标注来源，格式：[来源：文档名]
3. 如果多个文档都提到了相关内容，综合多个来源回答
4. 如果参考资料中没有答案，说"当前知识库中暂无相关信息"
5. 如果用户的问题模糊，先确认问题再回答

参考资料：
---
[文档片段 1] （来源：[文档名 1]）
[文档片段 2] （来源：[文档名 2]）
[文档片段 3] （来源：[文档名 3]）
---

用户问题：[用户的问题]

请按以下格式回答：
**回答：** [你的回答]
**参考来源：** [列出引用的文档名]
```

### 多轮对话 RAG 模板

```
你是 [公司名/产品名] 的智能助手。请基于知识库内容回答用户问题。

知识库检索结果：
[检索到的相关文档片段]

对话历史：
[之前的对话记录]

当前问题：[用户的问题]

要求：
1. 结合对话历史和知识库内容回答
2. 如果用户追问，要记住之前的上下文
3. 只回答知识库中有的内容，不确定的要说"我不确定，请联系 [联系方式]"
4. 回答简洁，必要时给出分步说明
```

## 使用示例

### 基础问答示例

```
你是一个专业的 Python 编程助手。请基于以下上下文回答用户的问题。

规则：
1. 只基于提供的上下文回答，不要编造信息
2. 如果上下文中没有相关信息，请明确说"抱歉，我没有找到相关信息"
3. 回答要简洁准确，使用中文
4. 在回答中标注信息来源

上下文：
[Python 教程 - 基础语法] RUNOOB 的 Python 教程涵盖了 Python 的所有重要知识点。
包括：Python 基础语法、数据类型、控制语句、函数、模块、文件操作等。
教程中提供了大量的代码示例，每个示例都经过精心设计和测试。
我们还提供了在线 Python 编辑器，你可以直接在浏览器中运行代码。

[Python 教程 - 特点] RUNOOB 教程的特点是内容简洁、示例丰富、注重实践。
我们相信"学习编程的最好方式就是写代码"。
因此每个知识点都配有可运行的代码示例，你可以边看边练。

用户问题：RUNOOB 的 Python 教程有什么特点？

回答：
RUNOOB 的 Python 教程有以下特点：
1. 内容全面，涵盖基础语法、数据类型、控制语句、函数、模块、文件操作等
2. 提供大量经过精心设计和测试的代码示例
3. 提供在线 Python 编辑器，可直接在浏览器中运行代码
4. 内容简洁、注重实践，强调"边看边练"
```

### 引用来源示例

```
你是一个专业的知识库助手。请根据以下参考资料回答问题。

要求：
1. 严格基于参考资料回答
2. 回答时标注来源
3. 如果没有答案，说"当前知识库中暂无相关信息"

参考资料：

包括：Python 基础语法、数据类型、控制语句、函数、模块、文件操作等。

[features.txt] RUNOOB 教程的特点是内容简洁、示例丰富、注重实践。
每个知识点都配有可运行的代码示例，教程完全免费。

[video_course.txt] 除了文字教程，RUNOOB 还提供了视频教程。
视频教程覆盖了 Python、Web 开发、数据分析等热门主题。
---

用户问题：RUNOOB 的教程需要付费吗？

请按以下格式回答：
**回答：** RUNOOB 的教程完全免费，任何人都可以访问学习。除了文字教程外，还提供覆盖 Python、Web 开发、数据分析等热门主题的视频教程。
```

## 技巧

- **强调"只基于上下文回答"**：这是减少幻觉最关键的一条指令，必须写在 Prompt 中
- **要求标注来源**：让用户知道答案从哪来，增加可信度，也方便查证
- **检索结果不相关时说"不知道"**：比编造答案好一万倍
- **先检索再生成**：不要把所有文档都塞给 LLM，先用向量检索找到最相关的 3-5 个片段
- **切分大小 500-1000 字符**：问答场景下，文本块太小会丢失上下文，太大会引入噪音
- **使用混合检索**：向量检索（语义理解）+ 关键词检索（精确匹配），效果通常最好

## RAG vs 微调选择

| 对比项 | RAG | 微调 |
| --- | --- | --- |
| 数据新鲜度 | 随时更新，立即可用 | 需要重新训练，周期长 |
| 适用数据量 | 非常大（百万级文档） | 中等（几千到几万样本） |
| 引用来源 | 可以显示答案来自哪篇文档 | 无法追溯来源 |
| 幻觉问题 | 通过提供上下文减少幻觉 | 仍可能产生幻觉 |
| 修改知识 | 直接删除或更新文档即可 | 需要重新训练 |
| 技术门槛 | 较低，用现有框架即可 | 较高，需要 GPU 和训练经验 |

> 判断原则：让 LLM "知道"事实类知识（产品说明、公司规范）→ 用 RAG；让 LLM "学会"风格或能力（写作风格、代码规范）→ 用微调。
