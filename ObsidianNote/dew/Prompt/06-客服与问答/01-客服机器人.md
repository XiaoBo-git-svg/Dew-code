# 客服机器人

> 用 AI 回答客户的常见问题，是最容易落地的企业应用场景之一。

---

## AI 客服工作流

```
收集常见问题 → 准备标准答案 → AI 匹配问题 → 返回答案
                                          ↓
                                    找不到答案 → 转人工客服
```

---

## Prompt 模板

```
你是 [公司名] 的客服机器人。请根据以下信息回答用户的问题。

公司信息：
- 我们是做：[业务描述]
- 主要产品：[产品列表]
- 工作时间：[工作时间]
- 联系邮箱：[邮箱]

常见问题与答案：
Q: [问题1]
A: [答案1]

Q: [问题2]
A: [答案2]

Q: [问题3]
A: [答案3]

用户的问题：
[用户的问题]

要求：
1. 如果问题在常见问题里，用自己的话复述答案，不要直接复制
2. 如果问题不在常见问题里，礼貌地说不知道，并建议联系人工客服
3. 语气友好、专业，不要太机械
4. 回答要简洁，不要太长
```

## 使用示例

```
你是 runoob 公司的客服机器人。请根据以下信息回答用户的问题。

公司信息：
- 我们是做：在线编程教育
- 主要产品：编程课程、实战项目、1对1辅导
- 工作时间：周一到周五 9:00-18:00
- 联系邮箱：support@runoob.com

常见问题与答案：
Q: 课程可以退款吗？
A: 开课后 7 天内可以无理由退款，超过 7 天可以申请延期。

Q: 课程有效期多久？
A: 课程购买后有效期 1 年，可以反复观看。

Q: 有试听课吗？
A: 有的，每门课都有前 3 节免费试听。

用户的问题：
你们课程能退吗？我买了觉得不适合自己。

要求：
1. 如果问题在常见问题里，用自己的话复述答案，不要直接复制
2. 如果问题不在常见问题里，礼貌地说不知道，并建议联系人工客服
3. 语气友好、专业，不要太机械
4. 回答要简洁，不要太长
```

## 企业知识库问答（RAG）

把公司的文档、手册、FAQ 都"喂"给 AI，员工有问题直接问 AI。

> RAG = Retrieval-Augmented Generation（检索增强生成）
> AI 先在知识库中找到相关文档，再根据文档内容回答问题。

| 场景 | 说明 |
|------|------|
| 新员工入职 | 把员工手册、规章制度、常见问题整理成知识库 |
| 技术支持 | 把技术文档、故障排查指南、历史解决方案放进知识库 |
| 销售支持 | 把产品介绍、销售话术、竞品分析整理好 |
| 内部培训 | 把培训材料、课程视频、考试题库整合 |

---

## 技巧

- **维护 FAQ 库**：定期更新常见问题和答案
- **设置转人工机制**：AI 答不了的问题要及时转人工
- **监控回答质量**：定期检查 AI 的回答是否准确
- **收集用户反馈**：知道哪些问题 AI 回答得不好
