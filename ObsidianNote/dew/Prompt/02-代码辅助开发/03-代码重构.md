# 代码重构

> AI 能帮你把"能跑的代码"变成"好代码"。

---

## Prompt 模板

```
请帮我重构这段代码，目标是：
1. 提高可读性
2. 减少重复
3. 更好的命名
4. 添加必要的错误处理

要求：
- 保持功能完全不变
- 给出重构说明：你改了什么，为什么这样改

原代码：
[你的代码]
```

---

## 使用示例

```
请帮我重构这段代码，目标是：
1. 提高可读性
2. 减少重复
3. 更好的命名
4. 添加必要的错误处理

要求：
- 保持功能完全不变
- 给出重构说明：你改了什么，为什么这样改

原代码：
def calc(p, t):
    if t == 'a':
        r = p * 0.03 * 1
        return p + r
    elif t == 'b':
        r = p * 0.04 * 3
        return p + r
    elif t == 'c':
        r = p * 0.05 * 5
        return p + r
```

## 重构方向速查

| 方向 | 说明 | Prompt 补充 |
|------|------|-------------|
| 可读性 | 变量命名、函数拆分、注释 | "重点改善变量命名和函数拆分" |
| 性能 | 算法优化、减少循环、缓存 | "重点优化性能，减少时间复杂度" |
| 可维护性 | 模块化、解耦、设计模式 | "用设计模式改善代码结构" |
| 健壮性 | 异常处理、边界检查 | "添加完善的异常处理和边界检查" |

---

## 技巧

- **明确重构目标**：不同目标会导致不同的重构方向
- **要求说明理由**：让 AI 解释为什么这样改，你能学到更多
- **保持功能不变**：强调这一点，防止 AI "顺便"改了功能
- **分步重构**：大段代码可以分模块逐步重构
