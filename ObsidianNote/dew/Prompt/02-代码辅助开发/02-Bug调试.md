# Bug 调试

> 遇到 Bug 先问 AI——这已经成为很多程序员的习惯。

---

## Prompt 模板

```
我遇到一个 Bug，请帮我分析可能的原因。

现象：[描述发生了什么，而不是你认为哪里错了]
期望结果：[应该发生什么]
实际结果：[实际发生了什么]

相关代码：
[相关的代码片段]

错误信息：
[完整的错误堆栈信息]

已经尝试过：
[你已经做过的排查]
```

---

## 关键原则

> **告诉 AI"发生了什么"，而不是"你觉得哪里错了"。**

如果你直接说"我觉得是数据库的问题"，AI 可能会顺着你的思路走，错过真正的原因。

### ✅ 正确做法

```
现象：用户注册时点击"发送验证码"按钮，页面显示加载中，但 10 秒后报错"请求超时"
期望结果：收到短信验证码
实际结果：页面报错，手机没收到短信
错误信息：TimeoutError: Request timed out after 10000ms
```

### ❌ 错误做法

```
我觉得是短信接口有问题，帮我看看
```

## 使用示例

```
我遇到一个 Bug，请帮我分析可能的原因。

现象：用户注册时点击"发送验证码"按钮，页面显示加载中，但 10 秒后报错"请求超时"
期望结果：收到短信验证码
实际结果：页面报错，手机没收到短信

相关代码：
async function sendCode(phone) {
  const res = await fetch('/api/send-sms', {
    method: 'POST',
    body: JSON.stringify({ phone })
  });
  return res.json();
}

错误信息：
TimeoutError: Request timed out after 10000ms
    at sendCode (register.js:45)

已经尝试过：
- 检查了网络连接，正常
- 用 Postman 直接调接口，可以正常返回
```

---

## 调试清单

在问 AI 之前，先准备好这些信息：

- [ ] **现象**：到底发生了什么？
- [ ] **期望**：应该发生什么？
- [ ] **代码**：相关的代码片段
- [ ] **错误信息**：完整的错误堆栈
- [ ] **已尝试**：你已经排查过什么

---

## 技巧

- **提供完整错误信息**：不要只截图，把文字复制出来
- **描述复现步骤**：怎么操作才能触发这个 Bug
- **说明环境**：操作系统、浏览器、语言版本等
