# 问题排查

> 遇到 Bug 不要慌，关键是把症状描述清楚。AI 像一个经验丰富的同事，你描述得越准确，它定位得越快。

---

## Prompt 模板

```
我遇到了一个问题：[问题描述]。

环境信息：
- 操作系统：[如：Ubuntu 22.04]
- 语言/框架版本：[如：Python 3.11, Django 4.2]
- 相关依赖：[如有]

错误信息：
[完整的报错信息/日志]

我已经尝试了：
1. [尝试过的方法一]
2. [尝试过的方法二]

请帮我：
1. 分析可能的原因（按可能性从高到低排列）
2. 给出对应的排查步骤
3. 提供解决方案
```

## 使用示例

### 数据库连接超时

```
我遇到了一个问题：API 接口间歇性返回 500 错误。

环境信息：
- 操作系统：Ubuntu 22.04
- 语言/框架版本：Python 3.11, FastAPI 0.100
- 数据库：PostgreSQL 15
- 部署方式：Docker Compose

错误信息：
sqlalchemy.exc.OperationalError: (psycopg2.OperationalError) 
connection to server at "db" (172.18.0.2), port 5432 failed: 
FATAL: too many connections for role

我已经尝试了：
1. 重启了数据库容器，暂时恢复，过一段时间又出现
2. 检查了代码里的数据库连接，每个请求都会新建连接

请帮我分析可能的原因和解决方案。
```

## 高效提问的结构

| 信息类型 | 为什么重要 | 示例 |
|----------|-----------|------|
| 问题现象 | 让 AI 知道发生了什么 | "页面加载后白屏"、"请求返回 500" |
| 错误信息 | 直接定位问题的关键 | 复制完整的报错堆栈，不要只截图 |
| 环境信息 | 排除环境差异 | Python 版本、OS、依赖版本 |
| 已尝试方法 | 避免 AI 重复建议 | "已经试过重启/清缓存/换浏览器" |
| 复现步骤 | 帮助 AI 理解触发条件 | "每次提交表单时出现" |

## 常见排查场景

### 环境问题排查

```
我的项目在本地能正常运行，但部署到 [服务器/容器] 后出错了。

本地环境：[版本信息]
部署环境：[版本信息]
错误信息：[报错内容]

请帮我分析可能是哪些环境差异导致的，并给出排查步骤。
```

### 性能问题排查

```
我的 [接口/功能] 响应很慢，需要排查性能瓶颈。

症状：
- 平均响应时间：[数据]
- 预期响应时间：[数据]
- 并发量：[数据]

相关代码：
[代码片段]

请帮我：
1. 分析可能的性能瓶颈
2. 给出排查和验证的方法
3. 提供优化建议
```

## 技巧

- **贴完整错误信息**：截图不如文字，截断的日志不如完整日志
- **描述"做了什么"到"出了什么问题"**：复现路径很重要
- **先搜后问**：把错误信息中的关键行告诉 AI，而不是"帮我看看"
- **一步步来**：先定位原因，再求解决方案，不要一步到位
