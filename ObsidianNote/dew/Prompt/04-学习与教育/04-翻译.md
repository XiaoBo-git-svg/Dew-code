# 翻译

> 翻译不只是字对字替换，更是文化和语境的转换。给 AI 设定专业角色，翻译质量会大幅提升。

---

## Prompt 模板

```
你是一位专业的 [源语言] 到 [目标语言] 翻译，擅长 [领域] 翻译。

请将以下内容翻译成 [目标语言]，要求：
1. 保持专业术语的准确性
2. 符合 [目标语言] 的表达习惯
3. 风格 [正式/口语化/文学化]

原文：
[你的内容]
```

---

## 使用示例

### 科技文档翻译

```
你是一位专业的英语到中文翻译，擅长科技文档翻译。

请将以下内容翻译成中文，要求：
1. 保持专业术语的准确性（如 API、SDK 等保留英文）
2. 符合中文技术文档的表达习惯
3. 风格正式，适合技术团队阅读

原文：
The new API endpoint supports batch processing of up to 1000 requests per second, with automatic retry on transient failures. SDK integration requires minimal configuration — just initialize the client with your API key and set the desired timeout.
```

## 翻译类型速查

| 翻译类型 | 关键要求 | Prompt 补充 |
|----------|----------|-------------|
| 技术文档 | 术语准确、保留专有名词 | "技术术语保留英文原文，首次出现时括号注释中文" |
| 商务邮件 | 语气得体、格式规范 | "符合商务邮件的格式和礼貌用语习惯" |
| 文学作品 | 保留风格、意境传神 | "保留原文的比喻和修辞风格，可以适当意译" |
| 口语对话 | 自然流畅、接地气 | "翻译成口语化中文，像日常聊天一样自然" |
| 学术论文 | 严谨准确、术语统一 | "保持学术论文的严谨性，术语前后一致" |

## 进阶用法

### 对照翻译（学习用）

```
请将以下英文翻译成中文，并附上逐句对照：

格式要求：
1. 英文原文
2. 中文翻译
3. 重点词汇/短语解释

原文：
[你的内容]
```

### 多版本翻译

```
请用三种不同风格翻译以下内容：
1. 正式书面版（适合公文/报告）
2. 日常口语版（适合聊天/社交媒体）
3. 简洁摘要版（一句话概括核心意思）

原文：
[你的内容]
```

---

## 技巧

- **指定领域**：同一个词在不同领域翻译不同（如 bug = 缺陷/虫子）
- **说明读者**：翻译给谁看，决定了用词的难易程度
- **要求对照**：学习场景下，逐句对照翻译更有助于理解
- **保留专有名词**：品牌名、技术术语通常不需要翻译
