# 博客写作流程

> 用一个完整的例子演示如何用 AI 写一篇技术博客：从大纲 → 分段撰写 → 整体润色。

---

## Prompt 模板

### 生成大纲

```
我想写一篇关于 [主题] 的文章，面向 [目标读者]。

请帮我生成一个大纲，包含：
1. 开头用一个 [类型] 场景引入
2. 3 个核心应用场景
3. 每个场景配一个具体例子
4. 结尾给出实际的行动建议
```

### 分段撰写

```
请帮我写 [某一部分]，用 [方式] 引入主题。

要求：
- 用具体的故事开头，不要空泛
- 场景要真实，让读者有共鸣
- 200-300 字

参考场景：[你的场景描述]
```

### 整体润色

```
请帮我通读全文，做以下润色：
1. 让段落之间的过渡更自然
2. 把太技术的术语用更通俗的方式表达
3. 确保文章风格统一
4. 检查是否有错别字或不通顺的句子

全文内容：
[你的完整文章]
```

---

## 使用示例

以写一篇"AI 如何帮助程序员提高效率"的博客为例：

**第一步：生成大纲**

```
我想写一篇关于"AI 如何帮助程序员提高效率"的文章，面向刚开始接触 AI 的开发者。

请帮我生成一个大纲，包含：
1. 开头用一个程序员日常痛点场景引入
2. 3 个核心应用场景（代码补全、Bug 调试、代码重构）
3. 每个场景配一个具体例子
4. 结尾给出实际的行动建议
```

**第二步：分段撰写（开头）**

```
请帮我写开头部分，用一个程序员的日常痛点场景引入主题。

要求：
- 用具体的故事开头，不要空泛
- 场景要真实，让程序员有共鸣
- 200-300 字

参考场景：周一早上，程序员小王发现上周五写的代码有 Bug，改了一上午还没找到原因，这时他想起可以用 AI 辅助调试...
```

---

## 技巧

- **大纲先行**：先让 AI 生成大纲，确认方向后再写正文，避免返工
- **逐段撰写**：长文章分段写，每段保持上下文清晰
- **统一润色**：所有段落写完后再整体润色，保持风格一致
- **AI 初稿 + 人工修改**：AI 生成的是初稿，你的修改才是灵魂
