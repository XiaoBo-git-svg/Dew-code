# 内容改写

> 同样的内容，面对不同读者需要不同的表达方式。AI 能帮你快速完成"一稿多改"，省去反复润色的时间。

---

## Prompt 模板

```
请帮我改写以下内容。

目标读者：[读者群体，如：技术团队/领导/客户/小学生]
风格要求：[风格描述，如：通俗易懂/专业严谨/轻松幽默]
字数要求：[字数，如：300 字左右/精简到一半]

改写要求：
1. 保留核心信息
2. 用更 [通俗/专业/简洁/详细] 的方式表达
3. [其他具体要求]

原始内容：
[你的内容]
```

---

## 使用示例

### 技术文档通俗化

```
请帮我改写以下内容。

目标读者：完全没有技术背景的公司管理层
风格要求：通俗易懂，避免任何技术术语
字数要求：200 字以内

改写要求：
1. 保留核心信息（做了什么、效果如何）
2. 用类比或日常语言替换技术概念
3. 突出业务价值而非技术细节

原始内容：
我们完成了微服务架构的迁移，将单体应用拆分为 12 个独立服务。
采用 Kubernetes 编排，实现了自动扩缩容。部署频率从每月 1 次提升到
每周 3 次，P99 延迟从 2s 降至 200ms，系统可用性从 99.5% 提升到 99.99%。
```

**改写示例：**
> 我们对系统进行了一次"大改造"——原来是一整台大机器，现在拆成了 12 个小模块各自运转。改造后，更新速度提升了 12 倍，用户操作的等待时间缩短了 90%，系统几乎不会再出现"宕机"。这意味着客户体验更好，业务迭代更快。

## 常见改写场景

| 场景 | 输入 | 输出方向 | Prompt 补充 |
|------|------|----------|-------------|
| 技术→商业 | 技术方案 | 突出业务价值 | "用投资回报率、效率提升来表述" |
| 长文→摘要 | 长篇报告 | 提炼核心要点 | "用 3-5 个要点概括，每个一句话" |
| 正式→口语 | 公文/通知 | 轻松易读 | "像朋友发微信一样表达" |
| 专业→科普 | 学术内容 | 大众能懂 | "多用类比和生活中的例子" |
| 中文→适配海外 | 中文内容 | 国际化表达 | "避免中国特有的梗和文化隐喻" |

## 风格改写对比

```
请用以下三种风格分别改写同一段内容：

1. 正式版：适合发给领导的报告
2. 口语版：适合团队内部沟通
3. 营销版：适合发在公众号/社交媒体

原始内容：
[你的内容]
```

---

## 技巧

- **先定读者再改写**：读者不同，表达方式完全不同
- **保留核心信息**：告诉 AI "哪些信息不能丢"
- **给出参考风格**：如果有喜欢的风格，可以附上示例让 AI 参考
- **分段改写**：长文逐段改写效果好于一次性改全文
