# 提示词基础结构

> 提示词的黄金法则：**输入模糊，输出也模糊；输入具体，输出才具体。**

---

## Prompt 模板

```
[角色设定]：你是一名 [专业身份]，擅长 [领域]。

[背景信息]：[描述任务的上下文和背景]

[任务描述]：请帮我 [具体任务]。

[输出格式]：[期望的输出格式，如：表格、列表、JSON 等]

[约束条件]：
1. [限制1]
2. [限制2]

[示例]（可选）：
输入：[示例输入]
输出：[示例输出]
```

> **四要素**：角色设定、背景信息、任务描述、输出格式。不必每次都写全，但写得越全面，结果越可控。

---

## 三种基础提示类型

### 问题型：直接提问

用于获取信息、解释概念、寻求建议。

```
Python 中 list 和 tuple 的区别是什么？
我是零基础，想做 Web 开发，应该先学 HTML/CSS 还是 JavaScript？请给出一个 3 个月的学习计划。
```

**关键**：越具体越好。不要只问"怎么学编程"。

### 指令型：告诉 AI 做什么

用于完成具体任务：写作、翻译、改代码、整理数据等。

```
把这段文字翻译成英文，保持原文的幽默感。
把这个长段落拆分成 5 个小点，每个点加一个小标题。
帮我优化这段代码，加上注释，并找出可能的 Bug。
```

### 对话型：多轮交流

适合复杂任务，一边聊一边调整方向。利用 AI 的上下文记忆，不需要每次都重复前面的内容。

```
你："我想做一个个人博客网站，有什么技术方案推荐吗？"
AI："根据你的需求，可以考虑以下几种方案：..."
你："我选 Hugo，那具体怎么开始呢？"
你："等等，我不想用 GitHub Pages，有没有其他免费的托管方案？"
```

## 使用示例

```
角色设定：你是一名有 5 年经验的前端开发工程师。

背景信息：我是一名大三计算机专业的学生，正在准备前端开发的面试，基础知识还行但缺乏实战经验。

任务描述：请帮我制定一个 4 周的前端面试准备计划。

输出格式：
1. 每周的学习主题和目标
2. 每天的具体任务（不超过 3 项）
3. 推荐的练习题类型

约束条件：
1. 每天学习时间不超过 3 小时
2. 重点放在 React 和 JavaScript
3. 包含至少 2 次模拟面试
```

## 好 Prompt vs 坏 Prompt 对比

| 维度 | ❌ 不推荐 | ✅ 推荐 |
|------|-----------|---------|
| 清晰度 | "帮我写点东西" | "帮我写一封给客户的邮件，内容是告知产品涨价 10%" |
| 角色设定 | "翻译这段话" | "你是专业的科技文档翻译，请翻译成中文，保持术语准确" |
| 格式要求 | "列个清单" | "用无序列表列出 5 个要点，每个要点不超过 20 字" |
| 上下文 | "帮我看看代码" | "这是 Python 爬虫代码，运行时报'连接超时'，帮我分析原因" |
| 风格语气 | "写个通知" | "写一份放假通知，语气正式，包含时间、注意事项、值班安排" |

> 好的提示词会告诉 AI：**你是谁、要做什么、给谁看、用什么格式、有什么约束条件。**

## 技巧

### 错误一：提示太模糊

| ❌ 模糊 | ✅ 具体 |
|---------|---------|
| "帮我写个文案" | "帮我写一篇奶茶店新品的微信公众号文案，目标读者 18-25 岁，风格活泼，突出'第二杯半价'，300-500 字" |
| "帮我看看代码" | "这段 Python 用户注册接口代码，帮我检查 SQL 注入风险、密码加密、错误处理，分别列出问题和建议" |
| "帮我做个计划" | "3 个月健身计划，目标减重 5 公斤，每周运动 3 次每次 1 小时，主要跑步和力量训练，按周安排" |

### 错误二：要求太多，一次说不完

复杂任务应该分成多轮对话，一步步来：

```
你："我想做一个电商网站，技术栈 React + Node.js + MySQL，第一步做什么？"
AI："建议先设计数据库表结构..."
你："好的，帮我设计用户表和商品表..."
```

### 错误三：不给反馈，一错到底

提示词是一个迭代过程：

```
你："这篇文章太长太技术化，读者是初学者。"
AI："抱歉，我来改简洁一些..."
你："好多了，但第二部分还是复杂，能用类比说明吗？"
AI："好的，用'寄快递'来类比这个网络通信过程..."
```
