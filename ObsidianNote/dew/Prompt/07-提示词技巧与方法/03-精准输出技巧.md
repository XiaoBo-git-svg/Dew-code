# 精准输出技巧

> 掌握这几个技巧，你的提示词功力会大增。

---

## Prompt 模板

```
[角色]：你是一名 [专业身份]。

[上下文]：[描述任务背景和场景]

[任务]：请帮我 [具体任务]。

[输出格式]：
- 格式：[表格 / 列表 / JSON / 代码 / 段落]
- 长度：[字数要求]
- 语气：[正式 / 轻松 / 专业 / 通俗]

[思维链]：请一步步分析，先说明推理过程，再给出结论。
```

---

## 指定输出格式

明确告诉 AI 用什么格式输出，能省去大量整理时间。

| 格式类型 | 示例指令 |
|----------|----------|
| 列表 | "用无序列表列出 5 个要点..." |
| 表格 | "用表格呈现，列名：功能、优点、缺点..." |
| 代码 | "输出完整的 Python 代码，包含注释..." |
| JSON | "输出 JSON 格式，字段：id、name、price..." |
| Markdown | "用 Markdown 格式，包含 3 个二级标题..." |

---

## 控制长度

告诉 AI 你希望输出多长，避免它说得太多或太少。

```
用一句话总结...
写一个 300 字左右的简介...
详细说明，至少 500 字...
每个要点不超过 20 字...
```

---

## 指定语气与风格

告诉 AI 用什么语气说话，适应不同的沟通场景。

```
语气正式，适合给领导看...
用轻松幽默的方式表达...
语言通俗易懂，适合小学生理解...
保持专业、客观、中立...
像朋友聊天一样自然...
```

---

## 思维链（Chain of Thought）

对于复杂问题，让 AI 把思考过程说出来，而不是直接给答案。这样可以显著提高答案的准确性。

```
请一步步分析这个问题，先说明你的思路，再给出答案。

遇到不确定的地方，请说明你的假设和推理过程。
```

**适用场景**：
- 数学/逻辑推理题
- 复杂方案的利弊分析
- 代码 Bug 的根因分析
- 多因素决策问题

---

## 使用示例

将多个技巧组合在一个提示词中：

```
你是一名有 8 年经验的产品经理。

我们的 App 日活从 10 万降到了 7 万，老板让我分析原因。

请用以下结构输出：
1. 可能的原因（表格形式，列：原因、可能性、依据）
2. 针对每个原因的解决方案
3. 优先级排序和行动建议

请一步步分析，先列出你的推理过程，再给出结论。

语气专业但不要太学术化，我需要明天在周会上汇报。
```

---

## 技巧

- **格式优先**：先想好要什么格式，再写提示词
- **长度要合理**：摘要要短、方案要长，别让 AI 猜
- **语气匹配场景**：给客户的和给团队的，语气完全不同
- **思维链用于复杂问题**：简单问题不需要，反而浪费时间
