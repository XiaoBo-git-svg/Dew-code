# 结构化输出

> 让 AI 返回 JSON、XML、Markdown 等结构化格式，方便程序处理——实际开发中最常用的技术之一。

---

## Prompt 模板

### JSON 输出模板

```
请把以下信息转换成 JSON 格式。

要求：
1. 只返回 JSON，不要任何其他解释文字
2. 字段名严格按照指定的名称
3. 数值类型不要加引号
4. 如果某个信息缺失，值设为 null

输出格式示例：
{
  "name": "姓名",
  "age": 年龄,
  "hobbies": ["爱好1", "爱好2"]
}

待转换的信息：
[你的文本]

JSON 输出：
```

### XML 标签模板

```
<system>你是 [角色]</system>

<task>[任务描述]</task>

<input>[输入内容]</input>

<output_format>
请严格按照以下 XML 格式输出：
<result>
  <field1>内容</field1>
  <field2>内容</field2>
</result>
</output_format>

<example>[一个完整的输出示例]</example>
```

---

## 使用示例

```
请把以下信息转换成 JSON 格式。
要求：只返回 JSON，不要其他文字。数值不加引号，缺失的字段值为 null。

待转换：李四，女，32 岁，在 RUNOOB 工作，喜欢读书和旅游，已婚

→ AI 输出：
{
  "name": "李四",
  "gender": "女",
  "age": 32,
  "company": "RUNOOB",
  "hobbies": ["读书", "旅游"],
  "married": true,
  "children": null
}
```

---

## 技巧

- **三要素**：明确字段名和类型 + 给一个示例 + 强调"只返回 XXX，不要其他文字"
- **XML 适合复杂任务**：用标签把 Prompt 分区（system/task/input/output_format/example），结构更清晰
- **长度控制**：简洁用"不超过 N 个字"，详细用"至少 N 字，分 X 点说明"，精确用"摘要 50 字，要点各 20 字"
- **数值类型**：明确要求"数值不加引号"，否则 AI 倾向于把所有值都当字符串
- **空值处理**：提前说明"缺失字段设为 null"或"返回空数组 []"，避免 AI 猜测填充
